User acceptance test cases

Product: CSR Organics storefront and internal admin tools (Karosale codebase).

Audience: QA, product owners, and operations — no technical background required.

How to use this document

  1. Fill in Environment URL below with your test site address (for example your preview or staging link).
  2. Run npm run seed once on that environment if you need sample products and orders (optional but recommended for packaging and pick-list tests).
  3. For each row, perform the steps in What you do, then check What you should see. Mark Pass or Fail in the last column and add the date and your initials.
  4. To share: send this HTML file as an attachment, or open it in a browser and use Print → Save as PDF. You can also copy the tables into Microsoft Excel or Google Sheets if your team prefers spreadsheets.

3. Storefront — shop, search, and sort order

# What you do What you should see Pass / Fail
S1 Open the Shop page. Do not type anything in the search box on the page yet. A grid of products appears. The page says how many products are shown out of the total (for example “Showing 24 of …”).
S2 In the shop search field, type a word you know appears in a product name (for example part of a seed name or “organic”). Click Filter. Only products that match that word in the name, short description, or SKU appear. The count at the top matches the filter.
S3 With a search still applied, open the Sort dropdown, choose Price: Low to High, then Filter. Products are listed from cheapest to most expensive. Order is stable and sensible on the first page.
S4 Choose Relevance as the sort (this is the default for many visits), enter a search term, and click Filter. Results still match your search. The order of products reflects “best match first”: closer name matches and stronger sales/ratings signals can appear higher than weak matches (exact behaviour is tuned in admin — see section 5).
S5 In the site header, use the Search products box (type at least two characters) and wait a moment. A small dropdown list of up to eight suggested products appears. Each suggestion shows name and price; clicking one opens that product’s page.
S6 Try a search term that matches no products. You get an empty product list (or a clear “no products” style message), not a broken page.

4. Admin tool — day-to-day operations

Sign in as admin.qa@csrorganics.com (after seed). Use the left-hand menu to move between sections.

# What you do What you should see Pass / Fail
A1 Open Dashboard. Cards for today’s sales value, today’s order count, orders still to fulfil, and low-stock product count. A short list of recent orders with customer names. A link toward Karma rewards management.
A2 Open Orders. Find an order and click View. Order number, customer contact, line items with quantities and money, current status badge, buttons for Print tag and shipping (where configured), and a way to change status and save.
A3 On a seeded packaging-ready order (SEED_DATA_PKG_ in notes if you can see notes in your tools), click Print tag. A PDF opens in a new tab (test data uses a public sample PDF). If generation is slow, the app may ask you to try again shortly — eventually a link should work.
A4 Change the order status in the dropdown, click Save status, then refresh the page. The status stays as you saved it.
A5 Briefly open Products, Inventory, Reviews, Karma Rewards, Marketing, Analytics, Customers, and Settings. Each page loads without a blank error screen. Content may be empty in a fresh environment but navigation works.

5. Search ranking confirmation — admin “Search & A/B”

This screen controls how strongly different signals influence product order when shoppers use Relevance sort or the header search suggestions. You are not editing product data here — only the “rules of the game” for ordering.

# What you do What you should see Pass / Fail
R1 Admin → Search & A/B. Wait for the page to finish loading. A form listing sliders or fields with plain-language labels (for example name match bonus, sales influence, rating, reviews count, featured/bestseller bonuses, in-stock bonus). A list area for experiments may appear below.
R2 Change one number slightly (for example increase “Featured product bonus” by a small amount), save the form. A success message such as “Saved.” Values remain after refresh.
R3 In the shop, run the same search twice with Relevance sort: once before your change, once after (you may need two browsers or a private window to compare if sessions differ). Product order may shift in a sensible way when featured products matter more (large changes make bigger shifts). If nothing moves, your catalogue may have few featured items matching that search — try another keyword or a bigger test change.
R4 Optional: create a small A/B experiment with two JSON variant configs (only if your team is comfortable with JSON), or skip. Experiment appears in the list; you can pause or resume it from the admin UI without errors.

6. Packer — today’s pick list

# What you do What you should see Pass / Fail
P1 Sign in as packer.qa@csrorganics.com. Open the packer pick list URL your team uses (commonly /packer/picklist). A page titled like “Today’s Pick List” with a count of line items. If the seed added packaging orders today, you see product names, SKUs, and quantities to pick.
P2 Tap Mark picked on one line. That row updates (picked quantity matches required, or row shows completed styling). List refreshes without a full page crash.
